# التوصيف الاكاديمي والتقني لمشروع Wino

آخر تحديث: 23-03-2026

## 1) التعريف بالمشروع
Wino منصة تجارة محلية رقمية (Local O2O Marketplace) تربط بين:
- تطبيق Flutter للعميل.
- Django + DRF API للخادم.

المبدأ البنيوي: **Store == User**.
اي مستخدم يمتلك ملفا موحدا يجمع هوية الحساب وبيانات المتجر.

---

## 2) الاشكالية البحثية
كيف نبني نظام بحث وتوصية محلي قابل للتفسير ويعمل بكفاءة تشغيلية ضمن:
- قيود موقع جغرافي واقعي.
- تنوع منتجات/اسعار.
- تفاوت ثقة البائعين.
- متطلبات قابلية التوسع مع كلفة تشغيل مناسبة.

---

## 3) الاهداف العلمية والتقنية
1. تصميم معمارية معيارية تفصل Presentation / API / Domain Logic.
2. تطبيق ترتيب متعدد العوامل للبحث المحلي.
3. بناء توصية سلوكية هجينة قائمة على سجلات تفاعل فعلية.
4. تفعيل تتبع تحليلي server-side مع حمل شبكي منخفض.
5. دمج monetization عملي (ads + wallet + subscription constraints).

---

## 4) المنهجية
تم اعتماد Prototype-driven Iteration:
- بناء مسارات العمل الاساسية اولا (users, catalog).
- اضافة analytics/recommendation تدريجيا.
- اضافة ads وwallet والاشتراكات كطبقة تجارية فوق النواة.
- تثبيت التحسينات عبر اختبارات تشغيلية وCI.

---

## 5) المعمارية العامة
### 5.1 backend
- Django 5 + DRF
- JWT auth (SimpleJWT)
- apps الاساسية:
  - users
  - catalog
  - ads
  - notifications
  - subscriptions
  - wallet
  - analytics

### 5.2 frontend (Flutter)
- Provider/ChangeNotifier
- RouteGenerator + named routes
- Firebase messaging/local notifications
- طبقات واضحة: core, data, features, presentation

### 5.3 integration
- REST API
- MEDIA uploads
- FCM

---

## 6) النموذج المفاهيمي للبيانات
### 6.1 كيانات رئيسية
- User (يتضمن بيانات المتجر)
- SystemSettings (اعدادات اقتصادية عامة)
- Category, Product, ProductImage
- Pack, PackProduct, PackImage
- Promotion, PromotionImage
- AdCampaign, AdCampaignViewer
- Follower, Review, Favorite, ProductReport, StoreReport
- CoinPurchase, CoinTransaction, CoinPackPlan
- InteractionLog, UserInterestProfile

### 6.2 قيود اتساق
- Review واحد لكل مستخدم/منتج (او متجر عند غياب المنتج).
- Favorite واحد لكل مستخدم/منتج.
- ملكية صارمة للتعديل (owner checks).
- Ad campaign يجب ان تستهدف اما product او pack فقط.
- SystemSettings singleton (سجل واحد عبر admin).

---

## 7) منطق ذكي منفذ فعليا
1. Adaptive radius في البحث المحلي حسب scarcity.
2. Weighted ranking (distance + price + reputation) مع تحسينات عملية.
3. توصية هجينة (category, price, popularity, recency, trust).
4. ads targeting:
   - audience_mode (all/followers/nearby/wilaya)
   - geo_mode (all/wilaya/radius)
   - age/category constraints
5. daily coin top-up:
   - عند GET /api/users/me/
   - اذا مر 24h والرصيد اقل من daily_login_coins يتم رفعه للحد
   - تحديث last_daily_coin_grant وlast_login

---

## 8) التحليلات
- InteractionLog يجمع الاشارات السلوكية.
- events endpoint متاح للدفع من التطبيق.
- جزء من الاحداث يشتق server-side من استخدام endpoints.
- ads clicks/impressions تدخل في دورة القياس التجاري.

---

## 9) الامان والحوكمة
1. JWT access/refresh + blacklist.
2. DRF permissions على مستوى ViewSet/APIView.
3. منع spoofing للمتجر في بعض flows (store=request.user).
4. تحقق صور الرفع (امتدادات/حجم) في wallet/subscriptions.

ملاحظة تنفيذية مهمة:
- اعدادات production hardening ما تزال مطلوبة (DEBUG, ALLOWED_HOSTS, CORS, SECRET_KEY env).

---

## 10) المكدس التقني
- Django + DRF
- Flutter
- SQLite (dev) / PostgreSQL (ready)
- Firebase Admin + FCM
- Celery + Redis
- Twilio OTP

---

## 11) KPIs مقترحة للتقييم العلمي/العملي
1. CTR/Conversion للتوصيات.
2. CTR/Conversion لحملات ads.
3. زمن استجابة البحث والتوصية.
4. retention للمستخدمين والتجار.
5. زمن الوصول لاول عملية مفيدة (first valuable action).

---

## 12) الحدود الحالية
1. التوصية rule/score-based وليست learning-to-rank كاملة.
2. المراقبة التشغيلية (APM/Sentry) غير مفعلة بالكامل.
3. بعض routes الاشتراكات في التطبيق تحتاج تطابقا ادق مع router الحالي.

---

## 13) افاق التطوير البحثي
1. Learning to Rank فوق النظام الحالي.
2. Embeddings للمنتجات/المستخدمين.
3. PostGIS عند التوسع الجغرافي.
4. A/B testing رسمي للبحث والتوصية والاعلانات.
5. Dashboard تحليلي لحظي للتاجر والمشرف.

---

## 14) خلاصة
Wino يقدم حالة متوازنة بين الطرح الاكاديمي والتطبيق الفعلي: 
- نموذج بيانات موحد وسهل التوسعة.
- دمج عملي بين local search + recommendation + monetization.
- قابلية عالية للتطوير نحو نسخة انتاج ناضجة للسوق الجزائري.
