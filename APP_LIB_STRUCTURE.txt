2) What each major folder is doing
----------------------------------
`core/`
- Cross-cutting infrastructure.
- API config, routing, providers, app services, theme, localization, utilities.
- Also now includes explicit accessibility helpers and a broader shared UI system.

Important files
- `core/config/api_config.dart`
- `core/config/app_config.dart`
- `core/constants/app_constants.dart`
- `core/routing/route_generator.dart`
- `core/routing/routes.dart`
- `core/services/api_service.dart`
- `core/services/storage_service.dart`
- `core/services/deep_link_service.dart`
- `core/services/external_maps_service.dart`
- `core/services/notification_badge_service.dart`
- `core/providers/locale_provider.dart`
- `core/extensions/l10n_extension.dart`
- `core/localization/runtime_translations.dart`
- `core/accessibility/semantic_helpers.dart`
- `core/theme/app_typography.dart`

`data/`
- Shared models and repositories used by screens/providers.

Important files
- `data/models/post_model.dart`
- `data/models/offer_model.dart`
- `data/models/pack_model.dart`
- `data/repositories/post_repository.dart`
- `data/repositories/store_repository.dart`

`features/`
- Reusable logic that is somewhat feature-oriented rather than screen-oriented.
- Currently active mainly in:
  - `features/analytics/`
  - `features/notifications/`

Reality note
- Older empty placeholders under `features/` were cleaned up.
- Most user-facing feature work still lives under `presentation/`.

`l10n/`
- Generated app localizations from ARB files.
- Current languages:
  - `app_en.arb`
  - `app_ar.arb`
  - `app_fr.arb`

Generated-file note
- Treat generated localization outputs as derived artifacts, not the primary place to edit strings.

`presentation/`
- Screen-first UI layer.
- This is where most real product work currently happens.

4) Routing pattern actually used
--------------------------------
- Active router: `RouteGenerator.generateRoute`
- Route registry: `core/routing/routes.dart`
- Navigation style: named routes + manual arguments
- `go_router` is installed but not the active navigation system

5) Localization pattern actually used
-------------------------------------
There are two localization layers:

1. Generated localization
- `l10n/*.arb`
- generated `app_localizations*.dart`
- Used for newer screens and strongly typed strings

2. Runtime translation bridge
- `core/localization/runtime_translations.dart`
- Used through `context.tr(...)`
- Helps migrate legacy hardcoded strings without blocking feature work

Important current rule
- Common/shared strings increasingly resolve through `AppLocalizations` first, then fall back to runtime translations for older paths.

6) Deep-link pattern
--------------------
- App bootstrap installs `DeepLinkService`
- Supported paths:
  - `/s/{id}`
  - `/p/{id}`
- Supported schemes:
  - `wino://`
  - `toprice://`

Main file
- `core/services/deep_link_service.dart`

7) State management pattern
---------------------------
- `Provider` + `ChangeNotifier`
- Main app providers are registered in `main.dart`
- Feature-specific API services are injected into some providers

Providers currently registered in `main.dart`
- `AuthProvider`
- `PostProvider`
- `HomeProvider`
- `LocaleProvider`
- `StoreProvider`
- `PackProvider`
- `WalletProvider`
- `AnalyticsProvider`

8) Notifications / app bootstrap notes
--------------------------------------
- App root widget: `WinoApp`
- Android local notification channel:
  - id: `wino_channel`
  - name: `Wino Notifications`
- Badge refresh and missed unread sync go through `NotificationBadgeService`

9) Platform scope
-----------------
- The repository is Android-only at platform-folder level.
- iOS, Web, macOS, Linux, and Windows folders were removed from the Flutter project tree.
- Some internal identifiers still use legacy names such as `dzlocal_shop`; this is an implementation detail, not the current public brand.

10) Practical editing notes
---------------------------
If you change one of these UI areas, inspect the related siblings too:
- product details <-> promotion details <-> pack details
- review/report UI <-> translations
- nearby/location UX <-> home/search/edit-profile
- deep links <-> routes <-> Android manifest
- merchant subscriptions <-> ads dashboard <-> wallet messaging
