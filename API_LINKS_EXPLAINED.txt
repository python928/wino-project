Base URL (from current app config)

Notes
-----
- Links below are relative API paths (prepend base URL).
- {id}, {storeId}, {productId}, {purchase_id} are dynamic values.
- Many list endpoints may return paginated `{results: [...]}` or direct lists depending on view settings.

1) AUTH
-------
- /api/auth/token/
  Method: POST
  Purpose: JWT login (username/email + password)

- /api/auth/token/refresh/
  Method: POST
  Purpose: refresh access token

2) USERS / PROFILE
------------------
- /api/users/register/
  Method: POST
  Purpose: create user and assign initial coins from `SystemSettings.first_login_coins`

- /api/users/auth/phone/send-otp/
  Method: POST
  Purpose: send OTP to phone (with rate limiting)
  Current dev/testing behavior: OTP is fixed and delivery is disabled in server code

- /api/users/auth/phone/verify-otp/
  Method: POST
  Purpose: verify OTP and login/create user (new phone users get first_login_coins)
  Current dev/testing behavior: verify expects the fixed code

- /api/users/me/
  Methods: GET, PUT, PATCH
  Purpose: profile read/update
  Extra behavior: GET triggers daily coin top-up check (24h logic + last_login update)

- /api/users/change-password/
  Method: POST
- /api/users/logout/
  Method: POST
- /api/users/preferences/categories/
  Method: POST

- /api/users/users/
  Methods: GET, POST, plus detail CRUD via router

- /api/users/followers/
  Method: GET
- /api/users/followers/toggle/
  Method: POST

- /api/users/store-reports/
  Methods: GET, POST, PATCH, DELETE (permission-dependent)

7) SUBSCRIPTIONS
----------------
Currently exposed in router:
- /api/subscriptions/merchant-subscriptions/
- /api/subscriptions/merchant-subscriptions/{id}/
- /api/subscriptions/merchant-subscriptions/access-status/
- /api/subscriptions/merchant-subscriptions/merchant-dashboard/

Important note:
- `subscriptions/views.py` contains `SubscriptionPlanViewSet` and `SubscriptionPaymentRequestViewSet`,
  but current `subscriptions/urls.py` only registers `MerchantSubscriptionViewSet`.
- If app uses `/api/subscriptions/plans/` or `/api/subscriptions/payment-requests/`, router registration must be added.

8) WALLET / COINS
-----------------
- /api/wallet/
  Method: GET
  Purpose: wallet snapshot (balances + transactions + costs)

- /api/wallet/packs/
  Method: GET
  Purpose: coin packs catalog

- /api/wallet/buy/
  Method: POST
  Purpose: submit coin purchase request with payment proof images

- /api/wallet/grant/
  Method: POST (admin)
  Purpose: manual coin grant

- /api/wallet/purchases/{purchase_id}/approve/
  Method: POST (admin)
  Purpose: approve pending purchase and credit user
