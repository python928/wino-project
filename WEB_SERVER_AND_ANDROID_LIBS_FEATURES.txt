4) Important backend capabilities
---------------------------------
- JWT auth with refresh rotation + blacklist
- Default authenticated API posture
- Filtering/search/ordering/pagination
- Scoped throttling for OTP, reviews, reports, analytics, feedback, payment flows
- Request correlation id in responses
- Trust/abuse scoring and moderation data
- Short deep links:
  - `/s/{id}/`
  - `/p/{id}/`

5) Important backend business rules
-----------------------------------
- `Store == User`
- `SystemSettings` controls signup/daily coin policy
- `TrustSettings` controls report/review abuse tuning and retention days
- Review uniqueness is enforced per user/target
- Report cooldowns and daily caps are enforced server-side

3) App architecture highlights
------------------------------
- `main.dart` bootstraps Firebase, local notifications, providers, deep links
- Active navigation is `RouteGenerator` + named routes
- State management is `Provider` + `ChangeNotifier`
- API access goes through `ApiService`
- Local language setting goes through `LocaleProvider` + `StorageService`
- Localization is hybrid:
  - ARB/generated strings
  - runtime translation bridge

4) Current app feature highlights

- wallet purchase flow

- notifications
- feedback screens

5) Deep-link / Android manifest highlights
------------------------------------------
Android manifest currently handles:

- `toprice://...`

6) App configuration notes
--------------------------
- Base URL is currently hardcoded in `api_config.dart`
- Notifications device endpoint used by the app:
  - `/api/notifications/devices/`
- Store APIs are aliased to users APIs because of `Store == User`

7) Product maturity summary
---------------------------
The codebase is no longer just a marketplace UI. It now has:
- local discovery
- trust moderation primitives
- merchant monetization
- feedback collection

- multilingual groundwork

This makes it technically rich enough for both real product iteration and academic analysis.
