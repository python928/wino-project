4) Important backend capabilities
---------------------------------
- JWT auth with refresh rotation + blacklist
- default authenticated API posture with selected public routes
- filtering/search/ordering/pagination
- scoped throttling for OTP, reviews, reports, analytics, feedback, payment flows
- request correlation id in responses
- trust/abuse scoring and moderation data
- short deep links:
  - `/s/{id}/`
  - `/p/{id}/`

5) Important backend business rules
-----------------------------------
- `Store == User`
- `SystemSettings` controls signup / daily coin policy
- `TrustSettings` controls report/review abuse tuning and retention days
- review uniqueness is enforced per user/target
- report cooldowns and daily caps are enforced server-side
- trust scoring weights seriousness and credibility rather than treating all reports equally

3) App architecture highlights
------------------------------
- `main.dart` bootstraps Firebase, local notifications, providers, deep links
- active app root widget is `WinoApp`
- active navigation is `RouteGenerator` + named routes
- state management is `Provider` + `ChangeNotifier`
- API access goes through `ApiService`
- local language setting goes through `LocaleProvider` + `StorageService`
- localization is hybrid:
  - ARB/generated strings
  - runtime translation bridge

4) Current app feature highlights
---------------------------------
- search + nearby + category filtering
- product / promotion / pack detail screens
- favorites and follow flows
- review / report flows
- share + QR flows
- merchant profile and publishing flows
- wallet purchase flow
- subscription plan and payment request flow
- merchant ads dashboard
- notifications
- feedback screens
- first-run language selection
- shared GPS/location education and settings recovery dialogs

5) Recent UX / product improvements that matter
-----------------------------------------------
- branding shifted in user-facing notification naming to `Wino Notifications`
- Android local notification channel id is `wino_channel`
- launch screen now includes visible language selection from first run
- nearby/location flows now use a shared helper for permissions, education, and settings recovery
- merchant ads dashboard now includes a clearer explainability card (“What these numbers mean”)
- common strings increasingly resolve through ARB localizations before runtime fallback
- report/review translations were expanded in Arabic and French

6) Deep-link / Android manifest highlights
------------------------------------------
Android manifest currently handles:
- `wino://...`
- `toprice://...`
- `http(s)://wino.app/...`
- `http(s)://toprice.app/...`
- `http://192.168.36.21/...`

7) App configuration notes
--------------------------
- Base URL is currently hardcoded in `api_config.dart`
- notifications device endpoint used by the app:
  - `/api/notifications/devices/`
- store APIs are aliased to users APIs because of `Store == User`
- subscriptions currently use `/api/subscriptions/payment-requests/` on the app side, with backend compatibility alias retained

8) Product maturity summary
---------------------------
The codebase is no longer just a marketplace UI. It now has:
- local discovery
- trust moderation primitives
- merchant monetization
- feedback collection
- deep linking and sharing
- multilingual groundwork
- onboarding / location education UX
- early explainability improvements for merchant analytics

This makes it technically rich enough for both real product iteration and academic analysis.
