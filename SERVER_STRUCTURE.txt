Important backend notes
-----------------------
- Active settings entrypoint: `app-backend/backend/settings.py`
- Active root router: `app-backend/backend/urls.py`
- `common/` is not a Django app in `INSTALLED_APPS`, but it is used by middleware/imports.
- Root short-link routes live in backend root:
  - `/s/{id}/`
  - `/p/{id}/`

Backend module roles
--------------------
- `users/`
  - auth, profile, OTP, followers, store reports, trust settings
- `catalog/`
  - categories, products, packs, promotions, reviews, favorites, product reports
- `ads/`
  - sponsored campaigns and click tracking
- `notifications/`
  - notification list/read flows + FCM device routes
- `subscriptions/`
  - plans, payment requests, merchant subscriptions, dashboard
- `wallet/`
  - coin packs, purchases, balances, approvals
- `analytics/`
  - recommendations, events, trust signals, interest profiles
- `feedback/`
  - user feedback and admin review flow

Platform scope
--------------
- Android platform folder is retained.
- iOS/Web/macOS/Linux/Windows folders were removed from the repo.

Key architecture notes
----------------------
- `Store == User` is the core backend domain rule.
- The Flutter app uses named routes via `RouteGenerator`.
- Localization is split between ARB-generated strings and runtime translations.
- The repo now contains both technical docs and memoire-oriented docs at the root.
