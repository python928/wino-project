# ما الذي ينقص المشروع الآن؟ (تقييم عملي محدث)
آخر مراجعة: 23-03-2026

## 1) نواقص حرجة قبل النشر
- الإعدادات أصبحت env-driven في `backend/settings.py` لكن القيم الافتراضية ما زالت development-friendly:
  - `DJANGO_DEBUG` default=True
  - `ALLOWED_HOSTS` مرن في وضع debug
  - `DJANGO_SECRET_KEY` fallback تطويري
- OTP ما يزال في وضع dev/testing (بحسب منطق الخدمات الحالي).
- غياب monitoring production متكامل (Sentry/APM/alerting).
- لا يوجد runbook مكتمل للطوارئ (keys rotation / DB restore / incident process).

## 2) ما تم تحسينه مؤخرا
- إضافة `feedback` app (API + screens).
- إضافة `trust-signals` endpoint وطبقة trust scoring.
- إضافة anti-abuse throttles/cooldowns.
- إضافة short links `/s/{id}` و`/p/{id}`.
- توحيد subscriptions routes: plans + payment-requests + merchant-subscriptions.
- تحويل Flutter لمنصات Android-only (حذف ios/web/desktop folders).
- تأسيس localization AR/EN/FR + `flutter generate` + l10n yaml.

## 3) نواقص جودة واتساق
- تغطية الاختبارات أقل من المطلوب مقارنة بحجم الميزات الجديدة.
- لا تزال نصوص UI كثيرة hardcoded وتحتاج إكمال migration إلى ARB.
- وجود Directionality/locale edge cases في بعض الشاشات يحتاج توحيد نهائي.
- بعض ملفات التوثيق كانت متأخرة وتم تحديثها بهذه الدفعة.

## 4) نواقص UX
- تحسين empty/loading/error states في wallet/ads/feedback وبعض تدفقات profile.
- تبسيط رسائل أخطاء OTP/Permissions لرفع الوضوح.
- تحسين first-run language experience قبل تسجيل الدخول.

## 5) نواقص التحليلات
- لا يوجد إطار KPI رسمي منشور داخل النظام (Precision@K/NDCG@K وغيرها).
- حاجة إلى dashboard متابعة أداء ads + recommendation impact.

## 6) أولويات التنفيذ (مرتبة)
1. Production hardening كامل + secrets policy.
2. Monitoring + alerting + log correlation.
3. توسيع الاختبارات الحرجة (Auth/OTP/Wallet/Ads/Subscriptions/Feedback).
4. إكمال localization coverage على كل presentation layer.
5. تحسين dashboard trader KPIs.

## 7) Definition of Done للإطلاق
- إعدادات أمان production مفعلة بالكامل عبر env.
- CI يمر بالكامل على كل PR.
- المسارات متطابقة ومثبتة باختبارات تكامل.
- Monitoring فعّال مع baseline alerts.
- توثيق تشغيلي محدث (runbook + recovery procedure).
